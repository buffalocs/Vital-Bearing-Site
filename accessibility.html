Accessibility

Health information should be understandable and usable.

Updated July 14, 2026. Vital Bearing is building Health OS for people with different vision, hearing, mobility, cognitive, and technology needs.

Health OS for iPhone

  • Supports VoiceOver labels and meaningful control descriptions.
  • Uses Dynamic Type and adapts key layouts for larger accessibility text.
  • Respects Reduce Motion and light or dark appearance preferences.
  • Does not rely on color alone for important status or meaning.
  • Uses plain language and avoids guilt-based feedback.

Vital Bearing website

  • Provides keyboard-visible focus, semantic headings, and a skip link.
  • Adapts to narrow screens, increased contrast, reduced motion, and dark mode.
  • Uses readable text sizing and does not require JavaScript for core content.

Known limitations

The private beta continues to undergo manual VoiceOver, large-text, contrast, and physical-device testing. Some imported documents may retain accessibility limitations from their original source.

Feedback

Private-beta members can use Settings → Help & Support → Beta Feedback to describe an accessibility barrier without automatically attaching health records.
